Developer Docs

Widget Embed Script

One script tag. Any website. Full analytics passthrough for GA4 and Meta Pixel.

1 Installation

Paste this before the closing </body> tag. The SMGBConfig block ensures your gym key is always picked up, even on tag managers where script attribute detection can be unreliable.

<!-- Set config before the script loads -->
<script>
  window.SMGBConfig = { gym: "YOUR_GYM_KEY" };
</script>
<script
  src="https://storage.googleapis.com/beta-assets/widget.js"
  data-smgb-gym="YOUR_GYM_KEY"
  crossorigin="anonymous"
></script>

No build step, no npm, no framework required.

2 CTA Buttons

Add data-smgb-widget to any button, link, or element. Elements added dynamically by JavaScript are picked up automatically.

<!-- Opens the Pass widget -->
<button data-smgb-widget="pass">Buy a Pass</button>

<!-- Events widget filtered to specific IDs -->
<a href="#" data-smgb-widget="event" data-smgb-show-only-events="42,43">
  View Classes
</a>

<!-- Calendar view, filtered templates, per-element gym override -->
<button
  data-smgb-widget="booking"
  data-smgb-gym="12"
  data-smgb-force-view="calendar"
  data-smgb-show-only-templates="5,7,9"
>Book Now</button>

Live demo — gym key 8

3 JavaScript API

Open widgets programmatically — useful for exit-intent flows, scroll triggers, or React/Vue components.

// Open the pass widget
SMGB.open("8", "pass");

// Open with filter options
SMGB.open("8", "booking", {
  forceView: "calendar",
  showOnlyTemplates: [5, 7, 9],
  hideHeader: true
});

// Close the overlay
SMGB.close();

5 Analytics Passthrough

Automatically collected and appended to every widget URL — no configuration needed.

SourceWhat gets forwarded
UTM / Click IDs utm_source, utm_medium, utm_campaign, utm_term, utm_content, fbclid, gclid, ttclid, msclkid, and more
GA4 _ga client ID from the _ga cookie, _gid session cookie
Meta Pixel _fbp browser fingerprint + _fbc click attribution cookie
Page context smgb_ref_url (current page URL), smgb_ref_title (page title)
postMessage Analytics payload sent into the iframe on load for in-widget attribution
dataLayer / gtag / fbq A smgb_widget_open event fired to GA4, Meta Pixel, and GTM on open

6 Global Configuration

window.SMGBConfig is read lazily when a widget opens, so it can be set anywhere on the page. Useful for switching between production and dev environments:

<script>
  window.SMGBConfig = {
    gym:     "8",
    baseUrl: "https://widgets-dev-kpe3ohblca-ew.a.run.app"  // omit for production
  };
</script>
<script src="https://storage.googleapis.com/beta-assets/widget.js"></script>
